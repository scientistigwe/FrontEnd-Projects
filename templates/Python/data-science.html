Scenario: Keyword Trend Analysis in CNN Articles


As a data scientist working for a research organization that analyzes large collections of CNN articles. The organization is interested in identifying and tracking the occurrence of specific keywords or phrases within these articles to understand trends, common topics, and emerging news themes.

For this task, you have been given a predefined set of keywords related to current events and emerging trends, including: "climate change" "pandemic" "election" "artificial intelligence" "economy" "technology" "healthcare"

Your organization has provided a dataset of CNN articles stored in a directory. Each row of the CSV file is an entity of an article with columns for Title, Description, Body, Keywords, Theme, and Link. You need to implement a high-performance string matching algorithm to efficiently search through this large text corpus and provide detailed reports on where these keywords appear.

Task

Your task is to develop a Python script that performs the following steps:

  1. Load a dataset of CNN articles.
  2. Construct an efficient string matching automaton using the flashtext algorithm with a predefined set of keywords.
  3. Search the text corpus using the automaton to find all occurrences of the keywords.
  4. Record the occurrences of each keyword, including their positions and the articles in which they appear.
  5. Generate a report that summarizes the findings, listing each keyword, its frequency, and the context around each occurrence.

Expected Outcome

A Python script that loads articles from a directory, constructs an keyword_processor with specified keywords, searches the articles for these keywords, and outputs a detailed report.

The report should list each keyword, the number of times it was found, and the positions and context within the articles.

Case Scenario: Environmental Monitoring System

Case Scenario: Environmental Monitoring System


A smart home technology company is developing a system to monitor and manage the living conditions within homes equipped with IoT devices. These devices measure various environmental factors such as temperature (temp), humidity, light intensity, presence of motion, smoke detection, and gas leaks (like LPG). The company aims to leverage these measurements to optimize energy usage, enhance comfort, and ensure safety. However, managing and interpreting the vast amounts of data generated by these sensors requires sophisticated data visualization techniques.

Objective:

The company intends to use Python and Chart.js to create a distributed data visualization framework that allows homeowners and technicians to view and interact with the sensor data in real-time. The goal is to provide clear insights into the current state of the home environment, detect anomalies early, and adjust settings automatically or suggest actions based on the data.

Task

Data Collection and Preparation: Preprocess the data to ensure accuracy and consistency, addressing issues such as missing values and duplicates.

Backend Development (Python):
Set up a Django backend to serve the air quality data through RESTful APIs.
Implement logic to aggregate and filter data based on device ID, timestamp, and other criteria.

Frontend Development (HTML/CSS/JavaScript):
Design a responsive web interface using HTML, CSS, and JavaScript.
Integrate Chart.js to display interactive charts and graphs that allow users to explore the data dynamically.

Chart Design and Implementation:
Utilize Chart.js to generate various chart types, such as line charts for temperature and humidity trends over time, bar charts for comparing light intensity across rooms, and heat maps for detecting high-motion activity zones.
Customize the charts with relevant labels, colors, tooltips, and interactivity features to enhance readability and usability.

Expected Outcome

By deploying this smart home environment monitoring system, the company will empower homeowners with real-time insights into their living environments. Technicians will also benefit from easy access to historical data for troubleshooting and optimization purposes. This system not only enhances the user experience but also contributes to energy conservation and safety improvements in smart homes.

About Dataset This task will use an open-source Hourly Air Quality data from the Automatic Urban and Rural Network for the years 2015-2023 (Oct) (see Kaggle Repo). The data covers several pollutants including Carbon Monoxide, Nitrogen Oxides, Nitrogen Dioxide, Nitric Oxide, Ozone, and Sulphur Dioxide. It also includes meteorological data for each hourly measurement, and also geospatial coordinates. The data includes monitoring from all AURN stations.
