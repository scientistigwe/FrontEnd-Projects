Scenario: Keyword Trend Analysis in CNN Articles


As a data scientist working for a research organization that analyzes large collections of CNN articles. The organization is interested in identifying and tracking the occurrence of specific keywords or phrases within these articles to understand trends, common topics, and emerging news themes.

For this task, you have been given a predefined set of keywords related to current events and emerging trends, including: "climate change" "pandemic" "election" "artificial intelligence" "economy" "technology" "healthcare"

Your organization has provided a dataset of CNN articles stored in a directory. Each row of the CSV file is an entity of an article with columns for Title, Description, Body, Keywords, Theme, and Link. You need to implement a high-performance string matching algorithm to efficiently search through this large text corpus and provide detailed reports on where these keywords appear.

Task

Your task is to develop a Python script that performs the following steps:

  1. Load a dataset of CNN articles.
  2. Construct an efficient string matching automaton using the flashtext algorithm with a predefined set of keywords.
  3. Search the text corpus using the automaton to find all occurrences of the keywords.
  4. Record the occurrences of each keyword, including their positions and the articles in which they appear.
  5. Generate a report that summarizes the findings, listing each keyword, its frequency, and the context around each occurrence.

Expected Outcome

A Python script that loads articles from a directory, constructs an keyword_processor with specified keywords, searches the articles for these keywords, and outputs a detailed report.

The report should list each keyword, the number of times it was found, and the positions and context within the articles.

Case Scenario: Big Data Analytics for Retail Company


A large retail company wants to implement a system to analyze their sales data from multiple stores in real-time. The sales data is extremely large and needs to be processed efficiently to gain insights quickly. The company aims to optimize their inventory management and enhance customer satisfaction by analyzing purchase patterns and trends.

Task

Implement a distributed file I/O system for parallel processing of large datasets to support real-time analytics for the retail company.

Data Ingestion: The system should ingest sales data from multiple stores into a distributed file system (HDFS).

Data Processing: Use parallel processing to process the ingested data in real-time, extracting useful insights such as sales trends, popular products, and inventory needs.

Data Analysis: Perform analytics on the processed data to produce reports and dashboards that can be used by management for decision-making.

Expected Outcome

Efficient Data Ingestion: The system should be capable of ingesting large volumes of sales data from multiple stores in real-time.

Actionable Insights: The system should provide actionable insights such as trends in sales, inventory requirements, and customer preferences.

Scalability: The system should be scalable to handle increasing data volumes as the company grows.

Fast Processing: Parallel processing should enable quick analysis of the ingested data.
